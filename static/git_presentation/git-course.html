...Git?

  • Desenvolvido em 2005, pela comunidade do Linux
  • Sistema de versionamento distribuído
  • Mantém a data em um diretório .git, em seu repositório

Commit

Um commit guarda informações como o seu parent commit, comentário e o autor (nome/email). Como todo objeto em Git, cada commit é identificado por um SHA.

Exemplo de um SHA: a30236028b7ddd65f01321af42f904479eaff549

Exemplo de uma branch simples

Repositórios são grafos de commits.

Exemplo com mais de uma branch

HEAD é a referência a sua branch atual.

git checkout -b feature_X
git checkout master

Commit com mais de um parent

Suponha que sua branch atual seja master.

git merge feature_X

GIT Url

Um repositório Git pode ser acessado de várias formas.

  • ssh
  • http[s]
  • git (protocolo utilizado para softwares open source)
  • local file

Um remote é um alias que se refere a uma URL de um repositório específico. Ao clonar um repositório, um remote chamado origin é automaticamente criado, que se referencia ao repositório clonado.
git remote show origin

Criando um novo repositório


git init
git add <filename>
git add -u *
git commit -m "Commit message"
git status


OBS: O comando -u permite que o git veja a sua branch atual como referência ao utilizar futuras operações.

Lifecycle de um arquivo

Adicionando um projeto existente a uma URL

git remote add origin <remote repository URL>
git push origin master

...ou você pode clonar um repositório já existente

git clone <remote repository URL>

Worflow básico de um projeto

git add <filename>
git commit -m "Fixed burritos bug"
git pull origin
git push origin <branch name>

Merge conflicts

Ocorre ao realizar merge em duas branches, ambas com arquivos modificados na mesma região.

Evite arquivos binários.

Utilizar .gitignore para não adicionar arquivos indesejados no repositório. 

							/nameOfFolderToIgnore/

# autogenerated files
*.userprefs
*.csproj
*.unityproj

# generated meta files
*.pidb.meta

ignoreThisSpecificFile.txt
Após um merge conflict, o git gerará um arquivo em um formato similar a esse: 
						<<<<<<< HEAD
Ed likes to eat glue.
=======
Ed likes to eat cupcakes.
>>>>>>> branchNameHere
Para resolver o conflito, decida qual bloco de texto deve permanecer (ou crie uma nova solução) e remova as áreas de marcação. Salve o arquivo e faça outro commit, voilà! 
						Ed likes to eat glue.

What now?

Há várias formas de usar o git: via linha de comando e via versões GUI, como SourceTree (Windows/Mac) e gitk (built-in GUI para Linux). Simples e fácil, basta verificar a instalação adequada para a sua plataforma.

Ter uma conta em um serviço online de hospedagem para repositórios git, aka repository hosting service.
Exemplos: GitHub, Bitbucket e GitLab.

Documentação simpática: git Doc.
